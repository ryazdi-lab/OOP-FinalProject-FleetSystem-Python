
 🚗 پروژه پایانی درس برنامه‌نویسی شیءگرا (OOP)
 
 سیستم مدیریت ناوگان و محاسبه هزینه‌ها (Fleet Management System)


 📖 صورت پروژه
در این پروژه، شما باید یک سیستم مدیریت ناوگان وسایل نقلیه را با رعایت کامل اصول برنامه‌نویسی شیءگرا در زبان پایتون پیاده‌سازی کنید. 
هستهٔ انتزاعی پروژه (`Vehicle`، `Rentable`، `Insurable`) از پیش در این مخزن قرار دارد و وظیفهٔ شما تکمیل بخش `concrete`، پیاده‌سازی رفتارهای خاص هر نوع وسیله نقلیه و اطمینان از کارکرد صحیح چندریختی است.

این پروژه صرفاً یک تمرین کدنویسی نیست؛ بلکه شبیه‌سازی یک گردش کار حرفه‌ای توسعه نرم‌افزار است که در آن باید از طریق `Fork`، `Branch` و `Pull Request` کد خود را به مخزن اصلی ارسال کنید.

---
ارسال پروژه فقط از طریق Pull Request به این مخزن پذیرفته می‌شود. ارسال فایل ZIP، ایمیل یا آپلود در LMS یا پیام رسان فاقد اعتبار است.

---
 🎯 اهداف آموزشی (۴ اصل OOP)
| اصل | انتظار از دانشجو |
| --- | --- |
| Encapsulation | استفاده از `@property`، اعتبارسنجی داده‌ها، جلوگیری از دسترسی مستقیم به متغیرهای داخلی (`_`/`__`) |
| Inheritance | ارث‌بری صحیح از `Vehicle`، استفادهٔ بهینه از `super()`، پرهیز از تکرار کد |
| Polymorphism | فراخوانی متدها از طریق آرایهٔ عمومی `list[Vehicle]` بدون استفاده از `isinstance` یا `type()` |
| Abstraction | پیاده‌سازی کامل متدهای `@abstractmethod`، تفکیک قرارداد از پیاده‌سازی، استفادهٔ صحیح از Mixinها |

 📤 نحوه ارسال پروژه (گردش کار گیت‌هاب)
 
✅ ارسال پروژه فقط از طریق Pull Request به این مخزن پذیرفته می‌شود. ارسال فایل ZIP، ایمیل یا آپلود در LMSیا پیام رسان فاقد اعتبار است.

1. این مخزن را Fork کنید.
2. مخزن Fork شده را Clone کنید و یک شاخهٔ جدید بسازید:
   ```bash
   git checkout -b feature/student-[شماره دانشجویی]
   ```
3. کدتان را پیاده‌سازی و تست کنید.
4. تغییرات را Commit و Push کنید (حداقل ۳ کامیت معنادار مثل کامیت های زیر):
   ```bash
   git add .
   git commit -m "feat: implement Car class and unit tests"
   git commit -m "refactor: extract validation logic to properties"
   git commit -m "test: add polymorphism coverage"
   git push origin feature/student-[شماره دانشجویی]
   ```
5. در گیت‌هاب یک Pull Request به شاخهٔ `main` این مخزن باز کنید.
6. در توضیحات PR، خلاصه‌ای از تصمیمات طراحی را قرار دهید.

 📜 قوانین و تعهدات آکادمیک
1. صداقت علمی: هرگونه کپی‌برداری از کد هم‌کلاسی‌ها، استفاده از کدهای تولیدشده توسط هوش مصنوعی بدون تغییر معنادار، یا خرید پروژه، منجر به نمرهٔ صفر خواهد شد.
2. ممنوعیت `isinstance` و `type()`: استفاده از این توابع برای تشخیص نوع شیء در لاجیک اصلی پروژه ممنوع است و نمرهٔ چندریختی را به صفر می‌رساند.
3. استاندارد کدنویسی: رعایت `PEP 8`، استفاده از Type Hints، نام‌گذاری معنادار و پرهیز از Magic Numbers الزامی است.
4. تعهد به Git Workflow: حداقل ۳ کامیت منطقی، پیام‌های کامیت به زبان انگلیسی و استفاده از Branch مجزا الزامی است. کامیت‌های `fix` یا `update` بدون توضیح  نباشند.
5. کمک گرفتن از LLM ها بلامانع است. اما باید به منطق کدتان مسلط باشید.
